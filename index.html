    Cofniecie - Twoja siatka bezpieczenstwa

    Cos sie zepsulo? Klikasz starsza wersje i ja przywracasz. Przywrocenie zmienia to, co widzisz teraz - ale historia zostaje nietknieta. Klik wersji pokazuje tez jej diff wzgledem poprzedniej.

    Repo = cala teczka, nie jeden plik

    Git nie pilnuje pojedynczego pliku. Pilnuje calej teczki projektu - wszystkich plikow naraz. Kazda zapisana wersja to zdjecie calej teczki.

    To Twoje repozytorium: folder ze wszystkimi plikami + historia jego wersji.

    Push i pull - wspolna polka

    Push kopiuje Twoje wersje na wspolna polke (na komputerze zostaja). Pull pobiera z polki cudze wersje. To wymiana kopii miedzy dwoma osobnymi miejscami, a nie wspolny ekran na zywo.

    Galaz i scalenie warto wiedziec

    Chcesz cos przetestowac bez ryzyka? Robisz osobny tor (galaz). Glowna wersja zostaje nietknieta. Gdy eksperyment sie udal, scalasz (merge) - na glownej powstaje nowy punkt scalajacy, ktory laczy oba tory.
